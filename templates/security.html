About Roles
  • Super Admin (Level 4) — Always has all permissions. Cannot be edited.
  • Admin (Level 3) — Can manage guests, hosts, users. Sees all guests regardless of tags.
  • Operator (Level 2) — Can SSH and apply updates on assigned guests.
  • Viewer (Level 1) — Read-only access to assigned guests.
Custom roles inherit a level from their base tier, which determines hierarchy. Users can only manage other users with a lower role level. Permissions on built-in roles (except Super Admin) can be customized.

About Tags
Tags control which guests non-admin users can see and interact with. Match tag names to Proxmox tags on your VMs/CTs. When discovery runs, guest tags are synced automatically. Admins can see all guests regardless of tag assignments.

Setup Guide

If you already have cloudflared running elsewhere (another CT, VM, or the Proxmox host), add a route to your existing tunnel:

  1. Open the Cloudflare Zero Trust dashboard > Networks > Tunnels
  2. Select your existing tunnel and click Configure
  3. Under Public Hostname, add a new entry:
    • Subdomain: e.g. lambnet
    • Domain: your domain
    • Service: http://{{ request.host.split(':')[0] if request.host else 'CT-IP' }}:5000
  4. Go to Access > Applications and create an application for the hostname
  5. Copy the Application Audience (AUD) tag and paste it above
  6. Configure Access Policies (e.g. allow specific email domains)

Install cloudflared directly on this CT and create a dedicated tunnel:

  1. Install cloudflared: bash setup.sh --cloudflared
  2. Authenticate: cloudflared tunnel login
  3. Create tunnel: cloudflared tunnel create lambnet
  4. Configure the tunnel to route to http://localhost:5000
  5. In Zero Trust dashboard, create an Access Application for your tunnel's hostname
  6. Copy the Application Audience (AUD) tag and paste it above
  7. Configure Access Policies (e.g. allow specific email domains)

Snapshot Policy
When enabled, a Proxmox snapshot is automatically created before non-admin users can apply updates or open SSH terminals. Snapshots are skipped if one was already taken within the last hour. Individual guests can override this with their own setting.
